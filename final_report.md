# Week 14 - Lab Sentiment Analysis: Final Report

## 1. Introduction
This report documents the sentiment analysis performed on a dataset of Facebook comments regarding a political topic (Rep. Zaldy Co). The goal was to analyze public sentiment using both Lexicon-based and Machine Learning (ML) approaches. The analysis was conducted using a Python-based pipeline within a Jupyter Notebook.

## 2. Methodology

### 2.1 Data Collection
The dataset consists of **1002 comments** collected from Facebook. The raw data contained mixed languages (Tagalog, Cebuano, English) and emojis.

### 2.2 Preprocessing
To prepare the data for analysis, the following steps were taken:
1.  **Deduplication**: Duplicate comments were removed to ensure unique data points.
2.  **Translation**: Non-English comments were translated to English using the `deep-translator` library (Google Translate API) to enable accurate analysis by English-centric tools like VADER and TextBlob.
3.  **Emoji Handling**: Emojis were converted to text descriptions (demojized) to preserve their sentiment value.
4.  **Cleaning**: URLs, special characters, and punctuation were removed. Text was lowercased.
5.  **Tokenization & Lemmatization**: Text was split into tokens, and stopwords were removed. Words were reduced to their base form using NLTK's WordNetLemmatizer.

### 2.3 Lexicon-Based Analysis
Two lexicon-based libraries were used to classify sentiment:
*   **VADER (Valence Aware Dictionary and sEntiment Reasoner)**: Optimized for social media text. It provides a compound score which was mapped to Positive, Negative, or Neutral.
*   **TextBlob**: A general-purpose NLP library. It provides a polarity score which was also mapped to the three sentiment classes.

### 2.4 Machine Learning-Based Analysis
Since the dataset was unlabeled, **VADER sentiment labels were used as the "ground truth"** to train the ML models. This allows us to evaluate how well ML models can learn to replicate the VADER classification logic.
*   **Feature Extraction**: TF-IDF (Term Frequency-Inverse Document Frequency) vectorization was used to convert text into numerical features (max 5000 features).
*   **Models**:
    *   **Logistic Regression**: A linear model suitable for text classification.
    *   **Naive Bayes (MultinomialNB)**: A probabilistic model often used as a baseline for NLP tasks.
*   **Evaluation**: Models were split into 80% training and 20% testing sets. Performance was measured using Accuracy, Precision, Recall, and F1-Score.

## 4. Comparative Analysis

### Lexicon Approaches (VADER vs. TextBlob)
*   **VADER** is generally more robust for social media text as it handles intensity and context better than TextBlob. In this dataset, VADER's ability to handle the translated text and potential remaining slang/informal language likely provided more accurate "ground truth" labels than TextBlob would have.
*   **TextBlob** is simpler but often less accurate for short, informal, or mixed-sentiment text found in comments.

### ML Approaches (Logistic Regression vs. Naive Bayes)
*   **Logistic Regression** consistently outperformed Naive Bayes across both targets.
*   **Naive Bayes** struggled significantly, especially with VADER labels (51.8% accuracy), likely due to its independence assumption failing to capture the nuanced phrases that VADER relies on.

### Learnability (VADER vs. TextBlob Targets)
*   Models achieved **higher accuracy when predicting TextBlob labels** (up to ~70%) compared to VADER labels (~68%).
*   This suggests that **TextBlob's sentiment logic is simpler** and more linearly separable based on word counts (TF-IDF), making it easier for models to replicate.
*   **VADER's logic is more complex** (handling negation, capitalization, punctuation), which is harder for a simple TF-IDF + Linear Model pipeline to capture fully without more advanced feature engineering.

## 5. Conclusion
The analysis successfully processed over 1000 comments. VADER provided a more nuanced sentiment distribution suitable for political discourse. In the ML experiment, **Logistic Regression trained on TextBlob labels** achieved the highest technical accuracy (~70%), but **Logistic Regression trained on VADER labels** (~68%) likely represents a more useful model for real-world application, as VADER itself is a better ground truth for social media text.
